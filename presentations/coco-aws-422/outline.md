### 判断（引出产品愿景）

> "我的判断是：**SaaS 的下一个形态不是 AI 功能加进软件里，而是 Agent 本身就是软件。**
>
> 我们内部叫它 SaaA — Software as an Agent。不是给软件加一个 AI 助手，是 Agent 自己就能完成你用软件完成的所有事。"

## 二、愿景：从 SaaS 到 SaaA（5 分钟）

### 三个层次的演进

| 层次 | 形态 | 例子 | 用户体验 |
|------|------|------|----------|
| 层次一 | 传统 SaaS | Salesforce, Jira | 人操作界面完成任务 |
| 层次二 | AI-enhanced SaaS | Notion AI, GitHub Copilot | 软件里嵌了一个助手 |
| **层次三** | **SaaA** | **COCO** | **你描述目标，Agent 端到端完成** |

### SaaA 的核心特征

> "一个真正的 SaaA 产品，用户不需要学操作界面。你不需要知道按钮在哪里、菜单怎么点。你只需要告诉 Agent：帮我监控这 10 个竞品的定价变化，每周出一份报告。它自己搞定数据源、搞定采集、搞定分析、搞定排版。
>
> 这不是未来五年的事。这是我们今天已经在卖的产品。"

### 为什么现在是时候

三个条件同时成熟：
1. **模型能力**：Claude / GPT-4 级别模型能真正理解复杂指令并执行多步任务
2. **基础设施**：AWS 的 Bedrock + Lambda + ECS 让 Agent 运行成本降到可商业化
3. **市场认知**：企业开始从"AI 能干什么"转向"我怎么让 AI 帮我干活"

## 三、产品：COCO 在做什么（7 分钟）

### 一句话

> "COCO 是一个 Agent 工作平台 — 让企业能快速部署、管理和扩展 AI Agent 团队。"

### 三层架构

```
┌─────────────────────────────────────┐
│  应用层：Agent Store + Workspace    │  → 用户看到的
│  (即用即走的 Agent + 自定义工作台)   │
├─────────────────────────────────────┤
│  能力层：Agent 引擎                 │  → 让 Agent 能做事
│  (记忆/通信/调度/工具调用/多模型)    │
├─────────────────────────────────────┤
│  基础设施层：Zylos Runtime (开源)   │  → 让 Agent 能跑
│  (持久运行/状态管理/安全隔离)        │
└─────────────────────────────────────┘
         ↕ 部署在 AWS 上 ↕
   Bedrock (模型) + ECS (运行时) + S3 (存储)
```

### 三个产品形态

**1. Agent Store（即用）**
- 预置 Agent：内容创作、数据分析、客服、代码助手
- 用户注册 → 5 分钟内 Agent 帮你完成第一个任务
- 按用量付费，0 门槛

**2. Agent Workspace（定制）**
- 企业客户搭建自己的 Agent 团队
- 自定义 Agent 角色、知识库、工具权限
- 多 Agent 协作：一个 Agent 做研究，另一个写报告，第三个做审核

**3. Zylos Runtime（自建）**
- 开源 Agent 运行时
- 给技术团队自己搭建 Agent 基础设施
- 持久记忆 + 多通道通信 + 任务调度 + 安全沙箱

### 差异化（vs 竞品）

> "市面上有很多 AI 工具，但大部分是两种模式：要么是对话框（ChatGPT），要么是工作流编排器（N8N、Dify）。
>
> 对话框的问题是：关掉窗口，它就忘了你是谁。
> 工作流编排的问题是：你要提前想清楚每一步，画出来，调试通。复杂场景画不完。
>
> COCO 不一样。我们的 Agent 有持久记忆、有自己的工具箱、能 7×24 持续运行、遇到意外能自己想办法。你不需要画流程图，你只需要描述目标。"

## 与 AWS 的联动点

既然是 AWS co-host 活动，slides 中可以自然提及：
1. **Bedrock**：COCO 底层用 Bedrock 调用 Claude 模型，AWS 原生集成
2. **ECS/Lambda**：Agent 运行时部署在 AWS 上，弹性伸缩
3. **S3**：Agent 知识库和记忆存储
4. **Security**：IAM + VPC 隔离，企业级安全合规

> 话术："我们选择在 AWS 上构建 COCO，是因为 Agent 需要的不是一张 GPU 卡，而是一整套基础设施 — 模型推理、持久存储、网络隔离、弹性扩展。Bedrock + ECS 这套组合，是我们评估过的最适合 Agent 运行时的平台。"
