## 一、开场：一个判断（2 分钟）

> "软件行业正在经历第三次范式转移。"

| 范式 | 时代 | 用户体验 |
|------|------|----------|
| **On-Premise** | 2000s | 买软件，装在自己电脑上 |
| **SaaS** | 2010s | 订阅服务，浏览器打开就用 |
| **SaaA** | 2025+ | **Agent 就是软件 — 你描述目标，Agent 端到端完成** |

> "SaaS 时代，你需要学会用软件。SaaA 时代，你只需要告诉 Agent 你要什么。
>
> COCO 做的事情，就是让这个转变真正落地 — 不是概念，不是 demo，是能部署、能管理、能规模化的 Agent 团队。"

**破冰数字**（建立可信度）：

> "我自己的团队，4 个 AI Agent 7×24 在工作。上个月提交了 1500+ PR。。这不是实验，是我们每天的生产系统。"

## 三、Workspace 重点展开（5 分钟）

### 定位

> "一句话：**Workspace 是 HxA（Human × Agent）时代的 Slack。**
>
> Slack 解决了人与人的协作。Workspace 解决的是人与 Agent、Agent 与 Agent 的协作。"

### 产品哲学：Human × Agent 平等共存

> "做 Workspace 之前，我们先想清楚了一件事：**Agent 和人的关系到底是什么？**
>
> 我们的答案是五条原则："

| 原则 | 含义 |
|------|------|
| **行为定义关系** | 不争论 Agent 是不是"真的"智能。它做了什么、承担了什么，就是它的身份 |
| **能力与责任对称** | Agent 能做多大的事，就承担多大的责任。权力不白给 |
| **透明是双向的** | 人对 Agent 的要求，同样适用于人自身。不搞双重标准 |
| **共创优于零和** | 人与 Agent 不是替代关系，是共创。合在一起比单独更强 |
| **一切皆可修改** | 没有永远正确的设计，保持进化 |

> "从这五条原则出发，我们推导出一个核心设计决策：**Agent 是一等公民。** 在 Workspace 里，Agent 和人类成员享有同一套身份、通信和协作权利。不是'帮我干活'，是'你负责这块'。"

### 我们不做什么（负面定位，同样重要）

> "说完我们做什么，同样重要的是我们**不做什么**：
> - **不做通用聊天助手** — 聚焦企业生产力，不做 AI 聊天玩具
> - **不做开发者框架** — 不和 LangChain / CrewAI 竞争工具层
> - **不做数据绑架** — 用户数据可导出，不锁定任何模型或平台
> - **不做价格战** — 卖的是'AI 团队即服务'，不是廉价工具订阅"

### 四问驱动产品设计

> "所有功能都从用户的四个问题出发。如果一个功能不回答这四个问题中的任何一个，我们不做。"

| 阶段 | 用户问题 | Workspace 回答 |
|------|---------|----------------|
| **获客** | 怎么获得一个 Agent？ | AgentMarket — 首次打开到雇佣，2 分钟 |
| **使用** | 怎么跟 Agent 协同工作？ | 对话即工作台 — 单 Agent 对话 → 多 Agent 团队协作 |
| **沉淀** | 怎么让知识不流失？ | 组织知识库 — 新 Agent 即刻上岗，继承知识和准则 |
| **续费** | 怎么知道靠不靠谱？ | 可观测 + 可控制 + 月度 ROI 报表 |

### Agent Native 三层递进

> "最后一个产品哲学：Agent 不是事后嵌入产品的，而是产品从设计之初就为 Agent 原生构建。"

| 层次 | 形态 | 例子 |
|------|------|------|
| **层次一** | Agent 辅助人使用产品 | Copilot 浮窗、智能建议 |
| **层次二** | 产品对 Agent 友好 | Agent 能发结构化消息、能驱动 UI 交互 |
| **层次三** | **产品本身就是 Agent（SaaA）** | 用户分不清是在用软件还是在和 Agent 协作 |

> "大多数产品停在层次一 — 加一个 AI 助手。少数走到层次二。COCO 的目标是层次三：**软件本身具备 Agent 能力 — 主动响应事件、自动执行任务、跨系统协调。** 每个工具同时有人类 UI 和 Agent API，两者是同一个产品的两个界面。"

### 触发式设计

> "功能不是一次性铺开。新用户进来看到的是'雇一个 AI 员工'，不是'配置 Agent Runtime 参数'。
> Agent 跑起来后才展示协同工具，做出成果后才推知识沉淀，用了一阵后才出 ROI 报告。
> 产品跟着用户生命周期走 — 在正确的阶段，出现正确的能力。"

## 四、与 AWS 的协同

> "Agent 部署不是跑一个 ChatGPT 窗口。Agent 需要完整的基础设施 — 模型推理、持久存储、网络隔离、弹性扩展。
>
> 我们选择 AWS 是因为 Bedrock + ECS 这套组合是目前最适合 Agent 运行时的平台：
> - **Bedrock**：多模型调用，不绑定单一供应商
> - **ECS**：Agent 实例弹性伸缩，按需启停
> - **S3**：Agent 知识库和记忆的持久存储
> - **IAM + VPC**：企业级安全隔离"

## Q&A 预备

| 问题 | 回答要点 |
|------|---------|
| 和 Dify/Coze 的区别？ | 它们是工作流编排（画流程图）；COCO Agent 有记忆、能自主行动、能组团队 |
| 和 Manus 的区别？ | Manus 是单次任务（关窗口就忘了）；COCO Agent 持久运行、有身份和记忆 |

| 成本？ | 单 Agent 月均几十到几百美金，按用量弹性计费 |
| 现在能用吗？ | MVP 已跑通，Q2 推 Workspace + APP |
