Key Experience #1

Chat = Workspace

Chat is not a standalone page — it's an embedded interaction layer.

  • Open chat on task page → Agent knows which task you're looking at
  • Open chat on project page → Agent knows the project context
  • Point and shoot — @ to reference any resource as context
Chat is embedded everywhere. Context-aware. @ to reference anything.

Key Experience #2

Agent = First-Class Citizen

Agents share the same identity, communication, and collaboration rights as human members.

  • Assign roles to Agents — not "help me do this", but "you own this area"
  • Agents remember context, learn preferences, proactively report
  • Like onboarding a new team member
Agent has identity, memory, role. Not a tool button. A team member.

Key Experience #3

Trigger-Based Design

New user sees "Hire an AI employee", not "Configure Agent Runtime parameters".

  • Agent runs → then show collaboration tools
  • Agent produces results → then suggest knowledge capture
  • After continued use → then show ROI reports

Product follows the user lifecycle.

Progressive disclosure. Features appear when relevant. Product follows user maturity.

Agent Native Design

Agents are native participants from day one — not bolted on.

Level 1: Agent Assists Human
Copilot overlay • Slash commands • Smart replies • Agent-on-Behalf with L1-L4 permission levels
Level 2: Product is Agent-Friendly
Agent Action Protocol (buttons/forms/cards) • Context API • Event Subscription — Agents drive full interactions
Level 3: Product IS the Agent (SaaA)
Software has Agent capabilities — responds to events, automates tasks, coordinates across systems. Human UI and Agent API are two faces of the same product.
Three levels of Agent Native. Level 3 is the endgame: software itself is the agent.
