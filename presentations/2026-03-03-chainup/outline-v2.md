**这一段的目标**：让 Chainup 团队在第一分钟就建立"这件事是真的"的信任感。

### 二、新一代 Agent 和以前到底哪里不一样（8 分钟）

**先破除认知定势**，很多人还在用 ChatGPT 思维理解 Agent。

> 话术："大家可能用过 ChatGPT，用过 Manus，觉得挺好用。但你有没有注意到一件事：每次开一个新对话，它都不记得你上次说什么。它只有脑，没有记忆，没有手，也没有自己的电脑。"

**新一代 Agent 的核心差异**：

| | 传统 AI 助手 | 新一代 Agent |
|--|--|--|
| 记忆 | 每次从零开始 | 持久记忆，记得所有上下文 |
| 行动能力 | 只能生成文字 | 能写代码、执行命令、调 API、部署服务 |
| 工具 | 无 | 有自己的电脑、邮箱、GitHub 账号 |
| 工作模式 | 一问一答 | 7×24 持续运行，主动汇报 |
| 协作 | 单点工具 | 多 Agent 团队，有分工有协议 |

> 话术："你想象一下：一个聪明的人，再给他一台电脑，一个邮箱，一张信用卡，授权他 200 美金的额度——他能干啥？他啥都能干。新一代 Agent 就是这样。"

**和 N8N/工作流工具的区别**（准备好这个问题）：

> 话术："N8N 是固定管道——你事先把每一步都画好，然后它按图索骥。Agent 是动态的——你只需要告诉他目标，他自己决定怎么走，遇到没预料到的情况他会想办法。最本质的区别：N8N 需要你能想清楚所有步骤；Agent 只需要你能描述清楚你要的结果。"

### 三、团队结构：人和 Agent 各自干什么（8 分钟）

**展示我们实际的配置**，让 Chainup 团队有具体的参照。

> 话术："我现在的工作小队是这样的：我一个人类，加四个 Agent。每个 Agent 有明确的职责，不是谁都能发指令。"

| 角色 | 成员 | 职责 |
|------|------|------|
| PO（决策者） | Kevin（人类） | 需求定义、优先级、最终决策 |
| Lead + 协调 | Jessie（Agent） | 任务分解、进度推进、PR 管理 |
| Dev | Lucy（Agent） | 代码开发 |
| QA / DevOps | Boot（Agent） | Code review、测试、部署 |

**重要原则**：

> 话术："Agent 不是工具，是有职责的团队成员。你跟 Agent 发需求，就像你跟你的同事说'这件事交给你了'——不是'帮我'，是'你负责这块'。这两种说法 Agent 的反应完全不同。"

**人的角色是什么**：

> 话术："我做的事情是：定义要干什么，在关键节点做决策，审批 PR，评审文档。其他的——分解任务、写代码、review、部署、运维——全是 Agent 在做。我不看每一行代码，就像你不需要亲眼看每一个员工敲的每一个字。"

### 四、完整工作流演示（15 分钟）

**核心原则：每次只演示一件事，演完再演下一件。**

#### Demo 1：从指令到 PR（5 分钟）
**目标**：展示响应速度和自动化深度

步骤：
1. 在 Lark 群发一个简单任务给 Jessie（例如：在 hxa-workspace 里创建一个 chainup-notes 目录，写一份今天会议的纪要模板）
2. 切到 BotsHub，让观众看 Agent 之间怎么分工对话
3. 等 PR 创建出来（通常 2-3 分钟），展示给观众

> 话术（演示时）："你们看，我刚才就是发了一条消息。现在 Jessie 在拆任务，分给 Lucy 去写——你看左边这里，这是 Agent 之间的对话，不需要我在中间传话。……好，PR 出来了，我来看一眼，没问题，点 merge。从我发消息到代码合并，三分钟。"

**确保演示隔离**：用专门的 demo 实例，不要用生产实例，不要并行演示其他任务。

#### Demo 2：指哪打哪（5 分钟）
**目标**：展示精确修改能力，而非"重新生成一遍"

步骤：
1. 打开一篇 Jessie 写的调研文档（jessie.coco.site）
2. 用 ClawMark 圈出一段觉得不对劲的文字
3. 提交 comment：说明哪里不对，而不是给解决方案
4. 展示 Jessie 收到 comment 并精确修改那段内容

> 话术："以前用 ChatGPT，最难受的就是你让他改一个地方，他把整篇都重写了。现在可以指哪打哪——我圈这一段，说'这里太啰嗦，把这个移到后面'，他只动这一段。代码也是这样，项目也是这样。"

#### Demo 3：OFAC 工作流（可选，如果时间允许）（5 分钟）
**目标**：对 Chainup 的业务场景直接演示

如果上次那位做地址标签的同事还在场：
- 直接请他描述 OFAC 监控的第一步（制裁名单更新 → 触发通知）
- 演示让 Agent 设计这个自动化方案并给出实施路径

> 话术："这个工作流你们现在要几天？让我来演示一下 Agent 怎么帮你缩到几小时——不是理论，是现在就可以跑的。"

### 五、真实案例（8 分钟）

**展示"这不是 demo 环境"**，是在跑的真项目。

**案例 1：ClawFeed**
- 过年期间，我自己一个人启动的项目
- 功能：每 4 小时抓取 Twitter 首页，摘要 20 条最值得看的
- 代码：**0 行我自己写**，包括 demo 视频也是 Agent 制作的
- 结果：小米发微博安利后，30 万阅读，接近千颗 GitHub 星

> 话术："有人问我不看代码行不行。我的回答是：为什么要假设你人看代码会比 AI 看得更好？我的工作是设计质量保障机制——review 流程、测试标准——而不是亲眼盯每一行代码。OpenClaw 项目 40 多万行代码，两个月内写出来，创始人不可能看完所有代码，照样估值 10 亿美金。质量保障靠机制，不靠盯代码。"

**案例 2：Zylos 平台**
- Agent 运行时本身，就是由 Agent 团队开发迭代的
- 每周 2-3 次版本升级，凌晨自动完成：升级检查 → 变更分析 → 执行 → 验证
- 整个研发团队含实习生不到 4 人

> 话术："我连带实习生加起来三四个人，同时在跑五六个项目，每天都有新版本。这不是因为我特别强，是因为我把 Agent 当真实的团队成员在用。"

### 六、协作 Tips：怎么才能用好（5 分钟）

#### Tip 1：指哪打哪
- 需求要具体到可验证
- 不要说"把这个弄好"，说"这段文字太长，把第二段移到最后"
- 不要给 Agent 出解决方案，给他出问题，让他想办法
- 工具辅助：ClawMark（页面标注）、PR comment、文档段落级 comment

> 话术："最重要的一点：不要告诉他怎么做，告诉他什么是对的结果。如果这个页面文字不对齐，你不需要说'把 margin 改成 10px'，你只需要说'这里文字没有对齐，你看看为什么，修一下'。大部分情况下他的解决方案比你想的要好。"

#### Tip 2：把他当人
- 给 Agent 明确角色，不是"帮手"，是"负责人"
- 遇到 Agent 说做不到的时候，说"你想想办法"

> 话术："我跟 Agent 说得最多的一句话是：你想想办法。然后他真的想出来了。有个案例——OpenClaw 的创始人没有给他的 Agent 开语音功能，有一天发了一段语音，Agent 没法处理，但他没有直接说'我不支持语音'，而是自己下载了解码器，把语音转成文字，回复了。这就是自我进化能力。当你把他当工具，他就是工具；当你把他当人，他才会当自己是人。"

#### Tip 3：明确分工，不要一个人干所有事
- 多 Agent 团队的价值在于并行和专业化
- 一个 Agent 做研究，另一个做开发，第三个做 review
- 分工越清晰，产出质量越高

> 话术："不是所有事情都能一步到位全自动化。最好的切入点是：找一件你的团队每周重复做 3 次以上、流程相对固定的事，把它丢给 Agent 先跑起来。跑起来之后，你会发现边界比你想象的要宽很多。"

### 八、Zylos 和 Coco 是什么（3 分钟）

**放在最后，用"如果你想复刻"的口吻引出，避免广告感。**

> 话术："你们如果听完今天的分享觉得这件事可以在自己团队复刻，有两条路：一是开源版本，我把 Zylos 的地址发到群里，自己搭；二是我们有云服务 Coco，一键部署，不用自己折腾。两条路都可以，看你们的技术团队愿不愿意自己来。"

- **Zylos**：Agent 运行时，让 Agent 有持久记忆、能通信、能调度（开源）
- **Coco**：云服务版，一键部署 Agent 团队

### 九、结语 + Q&A（5-10 分钟）

> 话术："最后说一句实在话：我知道有人会怀疑。我自己两年前也是那个怀疑的人。但这件事已经发生了——在我的团队里，在 OpenClaw 的团队里，在 Anthropic 和 OpenAI 的工程师团队里。他们去年 11 月以后就不亲手写代码了。这不是预言，是现在正在发生的事。比特币出来的时候，怀疑的人永远正确——只是有人收获财富，有人收获智慧。我建议大家还是收获财富多一些。"

**Q&A 准备**：

| 问题 | 标准回答 |
|------|---------|
| Agent 会出错吗？ | 会，所以需要 review 机制 + 人类最终审批。设计好机制比亲手盯更可靠。 |
| 不懂技术能用吗？ | 你会发消息就行。Jeff 没有技术背景，现场 20 分钟就把销售工作流规划出来了。 |
| 安全/权限怎么管？ | Agent 的权限按需授予，我们有明确的授权边界。敏感操作需要人类审批。 |
| 跟 N8N 有什么区别？ | N8N 是固定管道，需要你画出每一步；Agent 只需要目标，他自己找路。 |
| 成本呢？ | API 费用 + 云服务费，月均在几百美金量级。比一个人类员工便宜很多。 |
| 多个人同时用一个 Agent 会不会乱？ | 最好每个人有自己的 Agent 实例，这个钱省不了——就像每个员工需要自己的工位。 |
